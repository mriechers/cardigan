# v4.1 Sprint 3: User-Driven Retry — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace automatic tier escalation with a user-driven retry flow where users can pick a different model, add editorial notes, and see previous output + validation flags — all fed forward as context to improve the retry.

**Architecture:** The retry dialog becomes the primary quality-improvement mechanism. When a phase fails validation or errors out, the user sees what went wrong (validation flags, error messages, previous output) and decides how to fix it: different model, editorial guidance, or both. The retried phase receives enriched context (previous output + flags + notes), and the validator re-runs automatically after a successful retry.

**Tech Stack:** Python 3.13 / FastAPI, React/TypeScript, SQLite, pytest

**Design Spec:** `docs/superpowers/specs/2026-04-30-v4.1-pipeline-redesign-design.md` (Section 2)

## Task 5: Re-run Validator After Successful Retry

**Files:**
- Modify: `api/services/worker.py`

- [ ] **Step 1: Add validator re-run after successful retry**

In `retry_single_phase()`, after a successful phase retry (around line 433), if the retried phase is NOT the validator itself, re-run the validator:

```python
# Re-run validator after successful retry (unless we just retried the validator)
if phase_name != "validator" and phase_result.get("success"):
    try:
        logger.info("Re-running validator after retry", extra={"job_id": job_id})
        # Reload context with updated output
        context[f"{phase_name}_output"] = phase_result.get("output", "")
        validator_result = await self._run_phase(
            job_id=job_id,
            phase_name="validator",
            context=context,
            project_path=project_path,
        )
        if validator_result.get("output"):
            try:
                validation_data = self._parse_validation_result(validator_result["output"])
                await update_job(job_id, JobUpdate(validation_result=validation_data))
                logger.info(
                    "Post-retry validation complete",
                    extra={
                        "job_id": job_id,
                        "overall": validation_data.get("overall"),
                    },
                )
            except json.JSONDecodeError:
                logger.warning("Post-retry validator returned invalid JSON", extra={"job_id": job_id})
    except Exception as e:
        logger.warning(f"Post-retry validation failed (non-fatal): {e}", extra={"job_id": job_id})
```
